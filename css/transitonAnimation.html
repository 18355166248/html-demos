🎨 Transition vs Animation 动画演示

这个页面展示了 CSS Transition 和 Animation 的区别和用法。

Transition: 从一个状态到另一个状态的平滑过渡 | Animation: 可以创建复杂的多步骤动画序列

🔄 Transition 演示

Transition 用于在状态改变时创建平滑的过渡效果。 将鼠标悬停在元素上或点击按钮来触发动画。

🎭 Animation 演示

Animation 可以创建复杂的多步骤动画序列。 点击按钮开始/停止动画。

📊 Transition vs Animation 详细对比

特性 Transition Animation
触发方式 状态改变时自动触发 可以自动播放或手动触发
动画复杂度 简单的两状态过渡 复杂的多步骤序列
循环播放 不支持 支持无限循环
关键帧 只有开始和结束状态 支持多个关键帧
适用场景 悬停效果、状态切换 复杂动画、加载动画

💻 代码示例

Transition 示例：

.element {
  transition: all 0.5s ease;
}

.element:hover {
  transform: scale(1.2);
  background-color: #ff6b6b;
}

Animation 示例：

@keyframes bounce {
  0%, 20%, 50%, 80%, 100% {
    transform: translateY(0);
  }
  40% {
    transform: translateY(-30px);
  }
  60% {
    transform: translateY(-15px);
  }
}

.element {
  animation: bounce 1s infinite;
}

💡 使用建议

  • Transition 适用于：按钮悬停效果、表单验证状态、简单的状态切换
  • Animation 适用于：加载动画、复杂的交互动画、循环播放的动画
  • 两者可以结合使用，Transition 处理状态变化，Animation 处理复杂动画
  • 注意性能优化，避免同时使用过多动画效果
