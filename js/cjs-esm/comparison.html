CommonJS vs ES6 模块对比

深入分析两种模块系统的语法、特性和使用场景

CommonJS 模块

导出语法

// 单个导出 module.exports = value; // 多个导出 module.exports = { func1, func2, class1 }; // 动态导出 if (condition) { module.exports.debug = true; }

导入语法

// 导入整个模块 const module = require('./module.js'); // 解构导入 const { func1, func2 } = require('./module.js');

主要特性

  • 同步加载
  • 运行时确定
  • 支持循环依赖
  • Node.js 原生支持
  • 缓存机制
  • 动态导入（Node.js 12+）

ES6 模块

导出语法

// 命名导出 export const value = 42; export function func() {} // 默认导出 export default class {} // 重新导出 export { name } from './module.js';

导入语法

// 命名导入 import { func, value } from './module.js'; // 默认导入 import defaultExport from './module.js'; // 命名空间导入 import * as module from './module.js';

主要特性

  • 静态分析
  • 编译时确定
  • 树摇优化
  • 浏览器原生支持
  • 异步动态导入
  • 更好的错误处理

🔍 关键区别

加载时机： CommonJS 是同步加载，ES6 模块支持异步加载和静态分析。
确定时机： CommonJS 在运行时确定依赖关系，ES6 模块在编译时确定。
循环依赖： CommonJS 支持循环依赖，ES6 模块对循环依赖有更严格的限制。
优化能力： ES6 模块支持树摇优化，可以移除未使用的代码。
错误处理： ES6 模块提供更好的错误信息和调试支持。
